01  App Structure Overview  ·  Navigation Model & Tab Architecture

Navigation Model
The app uses a 5-tab UITabBar at the bottom of the screen — the primary navigation layer. All core sections are one tap away. Secondary navigation uses modal sheets and push navigation within each tab. No hamburger menus, no hidden drawers.

ADHD Principle: Tabs are always visible. Users never lose their place. Back buttons always work. No dead ends.

5-Tab Structure
Tab 1  🏠
Home — Daily dashboard, quick log, today's insight card
Tab 2  📚
Learn — Health topics library, search, saved articles
Tab 3  📊
Track — Symptom tracker, cycle log, mood & energy journal
Tab 4  🔍
Patterns — AI insights, correlation graphs, trend summaries
Tab 5  👤
Profile — Settings, health profile, subscription, goals

Screen Hierarchy
The overall structure follows 3 navigation levels maximum. No screen is more than 3 taps from any other screen.
Level 1
Tab root screens (5 total) — always accessible via tab bar
Level 2
Section screens — pushed from tab root (e.g., topic detail, tracker form)
Level 3
Detail screens — modal sheets or push (e.g., article, log entry review)

Max 3 levels deep. If content requires more, it signals a structural issue — restructure before building.

02  Onboarding Flow  ·  First-Run Experience · 5 Screens

Onboarding Design Principles
• Keep it under 3 minutes — ADHD brains disengage from long setups
• One question per screen — no forms, no overwhelming intake
• Show value immediately — don't save it all for after onboarding
• Skip always available — no forced completion
• Progress bar visible throughout — combats time blindness

03  Home Tab  ·  Daily Dashboard · Tab 1

The Home tab is the daily anchor point. It shows what matters today in a scannable, dopamine-friendly card stack. No infinite scroll. Maximum 5 cards visible before fold. Users can always see the tab bar.

This is the most-opened screen. Every element must load in <300ms. Cards must be skimmable in 5 seconds.

Home Screen Layout
Home (Today Dashboard)
Personalized daily summary. Card-based feed. Quick-log entry point.

▸ Navigation bar: 'Good morning, [Name]' + notification bell icon
▸ Card 1: Today's Phase — Cycle day + hormone phase label + 2-line description
▸ Card 2: Quick Log — 'How are you feeling?' with 5 emoji selectors (30-second log)
▸ Card 3: Today's Insight — Research snippet + source label (Evidence: High/Medium/Low)
▸ Card 4: Your Streak — Days tracked, gentle message (no shame if streak is broken)
▸ Card 5: Recommended Article — From Learn tab, based on current cycle phase
▸ Floating action button (+) in bottom-right for manual log entry

Quick Log Sheet (Modal)
Quick Log Bottom Sheet
Triggered by Card 2 tap or (+) FAB. Half-screen modal. 30-second interaction target.

▸ Sheet handle at top (standard iOS bottom sheet)
▸ Title: 'Today's Check-in' + date
▸ Row 1: Energy level — 5-tap scale (1–5, emoji icons, no numbers)
▸ Row 2: Mood — 5 emoji quick-tap (😰 😕 😐 😊 😄)
▸ Row 3: Focus — single slider 1–10
▸ Row 4: Any symptoms? — 6 chip tags (Cramps / Brain fog / Fatigue / Irritability / Bloating / Other)
▸ Optional note: 1-line text field (keyboard opens only if tapped)
▸ Submit CTA: 'Save Check-in' — full-width teal button

04  Learn Tab  ·  Health Content Library · Tab 2

The Learn tab is the content knowledge base. Articles are organized by health topic, not by recency. Topics use visual cards. All content shows evidence level. Reading time shown on every article.

No infinite scroll. Limit each topic section to 6 visible cards before 'See all'. Prevents overwhelm.

Learn Screen Layout
Learn (Library Root)
Topic-first browsing. Personalized section at top based on onboarding answers.

▸ Navigation bar: 'Learn' title + search icon (top right)
▸ Section 1: 'Based on your profile' — 3 horizontal-scroll cards
▸ Section 2: Health Topics grid — 7 topic cards (2-column grid, icon + label)
▸   Topics: Hormones & Cycle / ADHD / PMDD / PCOS / Perimenopause / Thyroid / Nutrition
▸ Section 3: 'Quick reads' — 3-card horizontal scroll, articles <5 min
▸ Section 4: 'Research highlights' — 2-card list, evidence-labeled research summaries
▸ Bottom: 'Saved articles' link → navigates to Saved screen

Topic Detail Screen
Topic Detail (e.g., Hormones & Cycle)
All content for one health topic. Organized into subtopic groups.

▸ Back button: returns to Learn root
▸ Topic header: large icon + topic name + 1-line description
▸ Evidence badge: 'High evidence base' / 'Emerging research' / 'Traditional practice'
▸ Subtopic chips: horizontal scroll — e.g., 'Follicular', 'Luteal', 'Menopause', 'ADHD link'
▸ Article list: vertical cards showing title / reading time / evidence level
▸ Each article card: 120pt height, title (2 lines max), reading time, bookmark icon

Article Detail Screen
Article Detail
Full content view. ADHD-optimized: headers, short paragraphs, key takeaway box.

▸ Navigation: back arrow + share icon (top right) + bookmark icon
▸ Reading progress bar: thin teal line at very top, fills as user scrolls
▸ Article header: topic badge + title (max 60 chars) + reading time + evidence level
▸ Key Takeaway box: purple callout at top — 2–3 sentence summary (ADHD skip-to feature)
▸ Body: H2 headers every 150 words max, short paragraphs, no walls of text
▸ Source citations: expandable footnotes at bottom, not inline
▸ Related articles: 2-card horizontal scroll at bottom
▸ Bottom CTA: 'Save to Your Library' (if not already saved)

Search Screen
Search
Triggered by search icon in Learn nav bar. Full-screen takeover.

▸ Search bar auto-focused (keyboard opens immediately)
▸ Recent searches: shown when field is empty
▸ Trending topics: 6 chip tags below recent (e.g., 'PMDD', 'Brain fog', 'Luteal phase')
▸ Results: article cards same style as topic list
▸ Empty state: 'No results for X — try a related term' + 3 suggested chips
▸ Cancel button: always visible, dismisses search and returns to Learn root

05  Track Tab  ·  Symptom & Cycle Tracker · Tab 3

The Track tab is the daily data entry hub. It combines cycle tracking, symptom logging, mood journaling, and custom health metrics. Entry is always fast — 60 seconds max for a full check-in. All historical data is reviewable here.

ADHD friction rule: If logging takes more than 3 taps for basic entry, users will stop. Design for 2-tap minimum.

Track Screen Layout (Root)
Track (Root)
Overview of current tracking status. Entry point for all log types.

▸ Navigation bar: 'Track' + calendar icon (top right)
▸ Today card: current date + cycle day + phase label (teal pill badge)
▸ Log buttons: 4 large tap-to-log pills (Mood / Symptoms / Cycle / Energy)
▸ Weekly mini-calendar: 7-day strip, dots showing logged days (purple = full, teal = partial)
▸ Recent logs: last 3 entries as compact cards, tap to expand
▸ Metrics section: 3 summary tiles (Avg energy this week / Symptom frequency / Current streak)

Cycle Log Screen
Cycle Log
Period start/end tracking + flow intensity. Simple 2-step entry.

▸ Back button: returns to Track root
▸ Month calendar view: color-coded days (period / predicted / ovulation window / luteal)
▸ Log period button: 'Start period today' or 'End period today' (context-aware)
▸ Flow intensity: 3 quick-tap options (Light / Medium / Heavy) — no sliders
▸ Symptoms for today: 6-chip multi-select (same as Quick Log sheet)
▸ Notes: optional 1-line text field
▸ Save CTA: 'Save Today's Log' — teal full-width button

▸ Section 4 — Physical symptoms: chip grid (12 options, scrollable row)

▸ Day detail sheet: shows all logs for that day as expandable cards

06  Patterns Tab  ·  AI Insights & Trend Analysis · Tab 4

The Patterns tab surfaces insights from logged data. It connects dots the user may not have seen: cycle phase + mood correlation, symptom clusters, energy trends. Available after 7+ days of tracking for basic insights, 30+ days for deep patterns.

Insights must be grounded in the user's actual data — never fabricated or generic. Each insight shows which data points it's based on.

Patterns Screen Layout (Root)
Patterns (Root)
Visual overview of detected correlations and trends. Scannable card layout.

▸ Navigation bar: 'Your Patterns' + share icon (export, premium)
▸ Insight card 1 (pinned): Most significant pattern this cycle — e.g., 'Energy dips on cycle day 21–24'
▸ Section: 'This cycle' — 3 trend cards (Mood trend / Energy trend / Symptom frequency)
▸ Section: 'Correlations found' — relationship insights (e.g., 'Low sleep correlates with high irritability')
▸ Section: 'Compare cycles' — graph showing last 3 cycles overlaid (premium feature, blurred for free)
▸ Unlock banner (free tier): 'Unlock deep patterns — see 6-month trends' → paywall

Insight Detail Screen
Insight Detail
Tap any insight card to see full explanation, data source, and recommendations.

▸ Back button: returns to Patterns root
▸ Insight headline: large, plain-language summary (e.g., 'Your energy peaks mid-cycle')
▸ Data basis: 'Based on 23 days of logs' — transparency about data source
▸ Chart: simple SwiftUI chart (bar or line) showing the specific metric
▸ Explanation: 2–3 sentence plain-language interpretation (no medical jargon)
▸ Research connection: 'Why this happens' — brief science note with evidence level badge
▸ Recommended actions: 2–3 ADHD-friendly action steps based on pattern
▸ Related articles: 1–2 cards from Learn tab on this topic

08  Authentication Flow  ·  Deferred Sign-Up Strategy

Deferred Sign-Up Strategy
Authentication is deliberately deferred until the first action that requires data persistence (e.g., saving an article, completing a second log). This reduces onboarding friction by ~60% and follows the ADHD-friendly principle of 'show value first, ask later.'

Users can explore the full app for up to 3 days without creating an account. Data is stored locally until sign-up. No account = no data loss warning shown after 72 hours.

Second day of tracking
Toast notification: 'Sign up to sync your health data across devices' → Sign up / Dismiss

Paywall trigger

10  Edge Cases & Empty States  ·  Handling Zero Data, Errors & Limits

Critical Empty States
New user / Day 1 Home
Welcome card replaces insight cards. 'Start your first check-in →' CTA prominent.
Track tab < 3 days
Mini-calendar shows only today. 'Keep logging — patterns appear at 7 days' progress bar.
Patterns tab < 7 days
Locked-looking card with progress bar: 'X more days to first insight'. Non-punishing framing.
Learn tab no connection
Cached articles shown first. 'Offline — showing saved content' banner. No error page.
Search no results
'No results for X — try: [3 related chips]'. Never a blank page.
Log history empty day
Tap empty day: 'Nothing logged that day — log today instead →' CTA.

Error States
Network error
Toast at bottom: 'Check your connection — we'll sync when you're back'. Never a full-screen error.
Sync failure
Subtle banner: 'Couldn't sync — your data is saved locally'. Auto-retries on reconnect.
Invalid form entry
Inline validation below field — red text, never alert popups.
Session expired
Redirects to Sign In with pre-filled email. Message: 'Your session expired — sign back in.'
Feature unavailable
Toast: 'This feature is temporarily unavailable'. No generic error codes shown.

Interruption Recovery
• App backgrounded mid-log: draft saved locally, restored when user returns
• Mid-onboarding close: progress saved, resumes from last completed step
• Payment interrupted: no double-charges, confirmation email always sent
• Article mid-read: reading position saved, returns to same paragraph on reopen

ADHD principle: Users will leave mid-task constantly. The app must gracefully handle interruption as the default state, not the exception.

12  MVP Priority Map  ·  Build Order & Scope

MVP Phase 1 — Core Screens (Ship First)
Target: 6–8 weeks to TestFlight beta with these screens complete.

1
Onboarding (5 screens)
No account required. Goals + health snapshot. Notification permission.
2
Home Tab — Root
Today card, Quick Log sheet, one insight card (hardcoded content initially)
3
Learn Tab — Root + Article
6 articles minimum, 2 topics. No search needed for MVP.
4
Track Tab — Root + Mood Log
Daily mood/energy log. Week mini-calendar. No cycle tracking in MVP.

Profile Tab — Root only
Health profile view. Notification toggle. No subscription screen in MVP.
6
Auth — Sign Up + Sign In
Apple Sign In + email/password. Deferred sign-up strategy active.

MVP Phase 2 — Add After Beta Feedback
1
Cycle tracking screens
Cycle log, phase detection, calendar view
2
Patterns tab
7-day insights. SwiftUI charts. Needs 7+ days of user data.
3
Search (Learn tab)
Full-text search across article library
4
Paywall + subscription
RevenueCat integration, annual/monthly plans, 7-day trial
5
Notification system
Push notification scheduling for daily check-ins

MVP Non-Goals (Version 2+)
• AI health correlation engine (needs data at scale)
• Export to PDF / CSV
• Apple Health integration
• Wearable device sync
• Community / social features
• Telehealth connection / provider sharing

Principle: Ship the daily logging loop and one full content topic before building anything else. Validate that users will log daily before building analytics on top of it.
