01  Color System
Every color in this system was chosen specifically for ADHD brains. No harsh reds for non-errors. No neon for attention. No shame-colors. Soft, purposeful, and calming.

Why ADHD Brains Need a Different Color System
Bright, high-contrast UI is visually stimulating and causes rapid fatigue in ADHD users. Our palette uses desaturated tones with clear semantic meaning — so color communicates purpose, not noise.

⚠  ADHD Color Anti-Patterns — Never Use These
	•	No red error states — use soft coral + clear text explanation
	•	No neon or overly saturated tones — causes visual fatigue
	•	No pure black (#000000) — use charcoal (#2D2D2D) instead
	•	No pure white (#FFFFFF) for backgrounds — use cream (#FFFEF9)
	•	No 'streak broken' red indicators — users abandon apps that shame them
	•	No grey text on white at small sizes — fails contrast (WCAG AA)

02  Typography Scale
Every type style is set for reading clarity on mobile. Line heights are generous. Font sizes stay comfortably large. No dense paragraphs anywhere in the app.

Primary Font: SF Pro Display (iOS) / Inter (Android fallback)
SF Pro Display is Apple's native system font — it is optimized for retina screens, ships built-in on every iPhone, and requires no additional loading. It is the correct choice for iOS ADHD UX: familiar, clean, and highly readable at all sizes.

Typography Rules for ADHD
	•	Maximum 3 type sizes per screen — prevents cognitive overload
	•	Body text minimum 16pt — never smaller for reading content
	•	Line height minimum 1.4x for body text — improves tracking
	•	Left-align all body text — center-aligned body is harder to scan
	•	Limit bold usage — use bold only for the most critical word, not the whole line
	•	No ALL CAPS for body content — use title case instead
	•	Maximum line length: 65 characters — prevents reading fatigue
	•	Paragraph max: 3 sentences — ADHD users stop reading after 3 sentences

Spacing Rule: Breathe More Than You Think
ADHD users experience visual noise acutely. When in doubt, add more space. Every card should feel like it has room to breathe. Tight packing = overwhelm = abandonment.

04  Component Library
Every component is designed to reduce decision fatigue, give clear affordances, and produce dopamine-friendly micro-feedback. No component should feel unclear or demand thought.

Cards

Content Card — Standard
Primary container for health information. One topic per card.
Background:  White (#FFFFFF) with 1px border (#E5E7EB) and 12px corner radius
Shadow:  0px 2px 8px rgba(0,0,0,0.06) — subtle depth, not dramatic
Size:  Full-width (minus 16px screen margins). Height: auto (content-driven)
Padding:  16px all sides
Topic tag:  Top-left. Lavender bg, Purple text, 6px radius, 11pt Micro font
Title:  Title Large (22pt Bold, Charcoal). Max 2 lines.
Body text:  Body (16pt, 24pt line-height, Charcoal). Max 3 sentences.
CTA:  Teal text link bottom-right: 'Learn More →' or 'Start Tracking →'
Evidence badge:  Bottom-left. Amber chip: 'Evidence: High / Medium / Low'
Tap state:  Scale to 0.97 + very slight shadow increase (200ms ease-out)

Insight Card — AI / Pattern Recognition
Surfaces connections between symptoms and cycles. Purple-accented.
Background:  Lavender (#EDE8F7)
Left accent:  4px solid Purple border on left edge
Icon:  ✦ or 🔮 icon in Purple (24pt), top-left
Header:  'Pattern Spotted' or 'Insight' in Purple, 13pt Caption, semibold
Content:  1–2 sentences max. Title Medium (18pt) — should feel like a discovery
Action:  Ghost button: 'See My History →' in Teal
Animation:  Fade + slide-up on first appearance (350ms ease-out)

Tracker Card — Daily Log
Quick-tap symptom and mood logging. Dopamine-friendly: instant visual feedback.
Background:  White with Mint (#E8F5F3) top accent strip (4px)
Tap zones:  Minimum 44×44pt. Generous spacing between options.
Input type:  Visual scale (1–5 with emoji) OR 3-option tap buttons (never text input)
Selection state:  Teal fill + white check icon + gentle bounce (200ms)
Label:  Short (2–3 words max). Title Medium (18pt)
Completion:  Full-card Teal flash → settle to Mint bg. Shows ✓ 'Logged!' in 0.5s
No-response state:  Mist background, no shame language (never 'You forgot to log')

Today Card — Daily Overview
The 'glanceable' card at the top of the Today screen. One key message.
Background:  Purple → Soft Purple gradient
Text color:  White for all text on this card
Content:  One headline (cycle phase or focus tip), 1 sentence context
Size:  Full-width, fixed height 140pt — always visible above fold
Icon:  Phase icon (moon, sun, leaf, fire) — 32pt, top-right
Progress ring:  Small cycle ring (32pt) showing phase position — bottom-right

Button Rules
	•	Maximum 1 primary button per screen — forces clear hierarchy
	•	No button text longer than 4 words — keeps CTAs scannable
	•	Corner radius: 12px for primary/secondary, 8px for ghost
	•	Loading state: Replace text with spinner + disable tap — never freeze silently
	•	Success state: Replace with '✓ Done' for 1.5s, then transition
	•	All buttons minimum 44×44pt touch target (WCAG AA + Apple HIG)

Motion Rules
Motion Type
Rule
Entry animations
Elements slide in from bottom (16px offset) + fade from 0→1 opacity
Exit animations
Fade to 0 opacity only — no sliding out (prevents nausea)
Scroll
Momentum-based iOS natural scroll. No snap scrolling unless intentional.
Loading
Purple pulse animation (1.5s loop, ease-in-out) — NOT spinning wheel when possible
Success feedback
Teal card flash → settle. Haptic: medium impact. Duration: 300ms
Error feedback
Gentle horizontal shake (3px offset, 250ms, 2 cycles) — NOT screen flash
Achievement
Confetti burst (Lavender + Teal dots, 600ms) + haptic: success pattern
Tab switch
Content crossfade (200ms) — no sliding between tabs
Reduced motion
All animations fall back to instant opacity change. Respect prefers-reduced-motion.
Looping animations
NEVER loop animations indefinitely — maximum 3 cycles then stop

Dark mode is not an inversion of light mode. Each color must be purposefully reconsidered for dark contexts. ADHD users frequently prefer dark mode — it reduces visual noise.

Dark Mode Rules
	•	Always use iOS adaptive colors (system colors) — they handle light/dark automatically
	•	Never manually switch colors based on interface style — use asset catalog color sets
	•	Shadows become lighter glows in dark mode (0% opacity shadows, replace with subtle borders)
	•	Purple accent stays slightly lighter (#9B7FD4) to maintain contrast on dark backgrounds
	•	Test every screen in both modes — do not assume automatic inversion looks correct
	•	Status bar: light content on dark backgrounds (automatic via iOS)
